# サバゲー動画作成（airsoft-highlight）

1日分のサバゲーPOV動画から、**戦闘シーン＋キルシーン**を自動で抜き出し、待機・単純移動を捨てて1本のハイライトに繋ぐ。

## 設計思想（なぜこの方式か）
- **戦闘＝音響特徴**。エアガンの連続発砲は高密度トランジェント＋音量スパイク。これで撃ち合いを検出する（映像解析より安定・高速）。
- **キル＝本人のコール**。当てた本人が「Nダウン」「やった」「ナイス」と発声しがち。Scribe転写の単語タイムスタンプから拾えば、**遠距離キルも音声で捕捉**できる（撃ち合いにならない単発キルは音響検出だけでは漏れるため）。
- **ハンドサインは当てにしない**。頭マウントのカメラだと挙手サインは画角外（上）に出て写らない。指数の自動判読は不可と判断済み。ヒット数キャプションが要るなら本人申告で受け取ること。

## 前提
- `ffmpeg` がPATHにあること（GoPro動画の音声抽出・カット・連結）。
- Python に `numpy` + `librosa`。`video-use` の venv を使うのが早い（環境に応じてパスを調整）:
  - Windows: `<video-use のルート>/.venv/Scripts/python.exe`
  - macOS/Linux: `<video-use のルート>/.venv/bin/python`
- **キル検出を使う場合のみ** ElevenLabs APIキー（`video-use/.env` の `ELEVENLABS_API_KEY`、Speech to Text 権限必須）。無い場合は戦闘検出だけでも成立する。
  - **クォータ実測（重要）**: Scribe STT は **約66クレジット/分** ≒ 無料枠10,000クレジットで**月150分程度**。1日分（約90〜100分）×2日でほぼ枯渇する。超過すると `401 quota_exceeded`。複数日や長尺を回す前にユーザーへ「クレジット消費（有料化の可能性）」を伝えること。枯渇時は転写できた分だけキル/除外を適用し、残りは**戦闘検出のみ**でフォールバック（その分はキル取りこぼし・退場混入が残ると明示）。
- スクリプトは `~/.claude/skills/airsoft-highlight/scripts/` にある。
- 作業ディレクトリは `<動画フォルダ>/edit/`（成果物はここに隔離。元動画は読み取り専用、絶対に削除・上書きしない）。

## 手順

最初に必ず**要件をユーザーに確認**する（毎回違う）: 完成尺の目安 / キル網羅の要否 / 会話シーンの扱い / BGM有無 / 縦横比。

```bash
# video-use の Python を指すように書き換える
# Windows例: PY="C:/Users/<you>/Developer/video-use/.venv/Scripts/python.exe"
# macOS/Linux例: PY="$HOME/Developer/video-use/.venv/bin/python"
PY="<video-use の Python パス>"
SK="$HOME/.claude/skills/airsoft-highlight/scripts"
V="<動画フォルダ>"          # 例: D:/サバゲー/202603xx あるいは $HOME/Videos/airsoft/2026-03-xx
W="$V/edit"; mkdir -p "$W"
```

1. **インベントリ**: `ffprobe` で各ファイルの尺・解像度・fps・音声を確認。GoProは4GB分割（`GH01xxxx`+`GH02xxxx`=連続録画）に注意。
2. **戦闘検出**（全ファイル）:
   `for f in $V/*.MP4; do "$PY" "$SK/detect_action.py" "$f" --out "$W/seg_$(basename ${f%.*}).json" --pct 68; done`
   - **ラベル付き運用では recall 下限を渡す**: ファイル名 `_N` の N を `--min-segments N` に渡すと、その試合で**最低N個の見せ場が出るまで閾値を自動で緩める**（ハンドサインは過小申告しがちなので、ラベルは「最低これだけは取れ」の下限として使う。余剰クリップは許容＝取りこぼし防止優先）。`_missing`/`_0` は下限なし。
3. **（任意・強く推奨）転写→キル＆除外検出**: 先に各ファイルを転写 →
   `"$PY" <video-use のルート>/helpers/transcribe.py <video> --language ja`（`$W/transcripts/<name>.json` に出力）。全ファイル済んだら:
   - キル: `"$PY" "$SK/detect_kills.py" --work "$W"` → `kills.json`（本人の「やった/ダウン/ナイス/当たった/取った/いただき」）
   - 除外: `"$PY" "$SK/detect_exclude.py" --work "$W"` → `exclude.json`（**復活カウントダウン**と**本人の被弾退場「ヒット通ります」**。これに重なるクリップは捨てる）
4. **組み立て**: `"$PY" "$SK/assemble.py" --work "$W" --target 700 --cap 90`
   （戦闘を尺target秒まで選抜＋キルは全採用、`exclude.json` 重複クリップは自動削除、撮影順整列。）
5. **レンダリング**: `"$PY" "$SK/render.py" --videos "$V" --work "$W" --out "$V/highlight.mp4"`
6. **品質レビュー（必須）**: 完成動画から数十フレームを抜いて目視。黒画面・破損・明らかな非戦闘・本人の被弾退場シーンが無いか確認し、あれば該当クリップを editlist から外して再レンダリング。

## チューニング
- 撃ち合いを増減 → `detect_action.py --pct`（高いほど厳選＝短い）。
- 完成尺 → `assemble.py --target`（秒）/ 1ラウンド偏り防止 → `--cap`（秒）。
- キルコールの語彙 → `detect_kills.py` の `PATTERNS`。プレイヤーの口癖に合わせる。
  「N人」「入った」は誤検出（敵発見・進入）もあるが、**取りこぼしより過剰採用が安全**なのでレビューで削る方針。

## 既知の落とし穴
- **日本語パスでの連結失敗**: ffmpeg の concat デマルチプレクサは concat リスト内の非ASCIIパスを文字化けさせる。`render.py` は相対ASCII名＋clipsディレクトリをcwdにして回避済み。
- **Scribeの日本語トークンは1文字単位**。複数文字フレーズ検索は単語結合してから（`detect_kills.py` で対応済み）。
- **長尺の電源スリープ**: 長時間処理中はWindowsがスリープして処理が止まりうる。ただし**電源設定の変更はユーザーの明示指示がある時のみ行う**（勝手に変更しない）。放置で長時間回す必要があり指示があれば `powercfg /change standby-timeout-ac 0` で一時無効化→完了後に必ず元値へ復元。指示が無ければ触らない。
- **キル誤検出の主犯**（実戦で判明）: ①復活カウントダウンの数字（「1,2,3…10」）が「N人」にマッチ ②実況/コール役の敵位置報告「A6入った」が「入った」にマッチ ③第三者の「ヒット」コール。→ キルパターンから「N人」「入った」「単独ヒット」を除外し、`detect_exclude.py` でカウントダウン＋退場を別途除外するのが必須。
- **「ヒット通ります」の転写ゆれ**: Scribeが「HITO取ります」「ひっておりまーす」「いております」等に化ける。`detect_exclude.py` の DEATH 正規表現で網羅。
- **最頻話者≠カメラ装着者の場合**: 近くに実況/コール役がいるラウンドでは dominant speaker がその人になる。話者IDだけで本人を特定しないこと。
- **「ナイス」は両義**: キル時だけでなく、自分が倒された時も相手を称えて言う。単独では信用せず、退場マーカーの除外ゾーンを手前10秒に広げて「被弾直前のナイス」を巻き込み除去する（`detect_exclude.py` 実装済み）。
- **ハンドサインのヒット数は過小申告しがち**（サイン=2でも実際は3ヒット以上取っている等）。重要な帰結：**ラベル数を「このラウンドはN個まで」という抽出の上限に絶対に使わない**。検出した戦闘/キル瞬間はラベル数と無関係に全部拾う（取りこぼし防止）。ラベルの用途は①字幕表示 ②0ヒット試合除外（`--drop-zero`）のみ。`--cap` は秒数上限でありヒット数とは無関係。字幕の数字もサイン由来のため実数より少ない可能性がある（必要なら検出キル数や「N+」表記も検討）。
- **キルコールは数えるためでなく「盛り上がる瞬間を探す」recall用**。過大にも過小にもなりうる申告で字幕やヒット数が狂わないよう、数の根拠とは切り離す。
